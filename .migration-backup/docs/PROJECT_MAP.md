## 5. Missing features

- Communication Hub is referenced as an upcoming focus but there is only a `/communication` page, not a fully built multi-channel conversation platform.
- VoIP calling, AI transcription, and call recording are described in `docs/PROJECT_CONTEXT.md` as planned but are not implemented in the current code.
- AI Meeting Assistant is listed as upcoming but no workflow or API route exists for meeting summarization/assistant capabilities.
- External enrichment integration is stubbed in `app/api/enrich/route.ts` with a TODO for Clearbit/Apollo.
- Payment provider integration is disabled in `app/api/revenue/payment-intent/route.ts` pending Stripe/Razorpay API keys.
- Client portal workflows exist in components, but there is limited evidence of a dedicated client-facing routing or access control implementation beyond `app/client/[token]/page.tsx`.
- HR onboarding and recruitment have placeholder/pending values for candidate fields and employee status, indicating incomplete end-to-end hiring activation.
- Attendance and leave workflows are partially implemented, with commented phase-0 schema logic in `app/team/attendance/page.tsx`.
- There is no visible test suite or test files in the current workspace.
- There is no explicit Supabase migration folder or documented database schema in the repository root.

## 6. Technical debt

- Multiple `TODO` comments remain in business logic, including enrichment provider integration and payment intent reactivation.
- Several pages use broad `any` types (`useState<any[]>`, untyped API response handling), increasing risk in TypeScript maintenance.
- Many UI pages contain repeated Supabase query logic instead of a shared data access layer.
- Placeholder values like `PENDING` for candidate contact fields and incomplete HR onboarding status flows are still present in production logic.
- Route and API naming patterns are somewhat irregular (`followups` vs `follow_ups`, `crm_todos` vs `todos`), which may create confusion over table naming and API conventions.
- The codebase contains large page components with deeply nested client-side state and fetch logic, suggesting a need for modular refactor and component extraction.
- No automated tests are visible, so regression coverage is likely absent.
- `hr-docs` is used as a storage bucket without explicit documentation of bucket rules or schema.
- The app appears to rely on business-specific hardcoded status strings across multiple features, making refactor and state transitions brittle.

## 7. Recommended roadmap

### Immediate (0-3 weeks)
- Add internal documentation for Supabase schema and row-level security expectations.
- Consolidate Supabase access into shared library functions and remove repeated query logic from page components.
- Clean up `TODO`/placeholder code paths in `app/api/enrich/route.ts`, `app/api/revenue/payment-intent/route.ts`, and HR pages.
- Introduce a small test suite focused on page rendering and API route results.
- Audit status strings and normalize them across CRM and HR features.

### Mid-term (3-6 weeks)
- Implement full Communication Hub capabilities, including unified messaging, WhatsApp integration, and multi-channel conversation routing.
- Build VoIP/AI transcription/call recording support in the communication stack once infrastructure decisions are made.
- Harden the client portal experience with authenticated access, invoicing, approvals, and project updates.
- Complete HR workflows for recruitment, onboarding, attendance, leave approvals, and payroll with proper validation.
- Integrate a real enrichment provider and live payment gateway support.

### Strategic (6-12 weeks)
- Create a multi-tenant architecture plan for Supabase and hosted workspaces.
- Add role-based access control and permission enforcement across APIs and pages.
- Develop analytics and revenue intelligence dashboards with forecasted pipeline metrics.
- Build a reusable, documented component library for cards, modals, tables, and portal widgets.
- Establish continuous integration with build checks and automated regression tests.
